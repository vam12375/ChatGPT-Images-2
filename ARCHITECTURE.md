## 数据流程详解

### 1. 请求流程

```
客户端请求
    ↓
┌─────────────────────────────────────┐
│ POST /api/images/generate           │
│ {                                   │
│   "prompt": "图片描述",             │
│   "size": "1024x1024",              │
│   "quality": "high",                │
│   "outputFormat": "png",            │
│   "count": 1                        │
│ }                                   │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 1️⃣ 验证请求参数                    │
│    - 检查 prompt 非空               │
│    - 验证 size、quality 有效值     │
│    - 限制 count 在 1-4 之间         │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2️⃣ 生成缓存键                      │
│    MD5(prompt + size + model)       │
└─────────────────────────────────────┘
    ↓
        ┌──────────────────────┐
        │ 缓存中有该请求吗？   │
        └──────┬───────────────┘
        是     │      否
        │      │
        ▼      ▼
    返回缓存   ┌─────────────────────────────────┐
    结果       │ 3️⃣ 获取下一个 API 密钥          │
              │    支持自动轮转多个密钥          │
              │    负载均衡，防止单点故障        │
              └─────────────────────────────────┘
                    │
                    ▼
              ┌─────────────────────────────────┐
              │ 4️⃣ 调用 OpenAI API              │
              │    model: dall-e-3              │
              │    response_format: b64_json    │
              └─────────────────────────────────┘
                    │
           ┌────────┴────────┐
           │ 请求成功？       │
           └────────┬────────┘
           成功     │      失败
           │        │
           ▼        ▼

        响应       │ 重试逻辑                │
           │       │                         │
```

## 组件设计

### ProxyConfig (API 密钥管理)

```javascript
ProxyConfig
├── apiKeys[]           // API 密钥数组
├── currentKeyIndex     // 当前轮转位置
├── requestCount{}      // 每个密钥的请求计数
│
├── parseApiKeys()      // 从环境变量解析密钥
├── getNextApiKey()     // 负载均衡获取下一个密钥
├── getRequestStats()   // 获取密钥统计信息
└── getStats()          // 获取全局统计信息
```

### ProxyMiddleware (缓存和日志)

```javascript
ProxyMiddleware
├── cache{}             // 内存缓存存储
├── requestLog[]        // 请求日志（最近100条）
├── maxCacheAge         // 缓存过期时间
│
├── generateCacheKey()  // 生成 MD5 缓存键
├── getFromCache()      // 从缓存获取数据
├── setCache()          // 存入缓存
├── logRequest()        // 记录请求
├── clearCache()        // 清空所有缓存
└── getCacheStats()     // 获取缓存统计
```

### ProxyHandler (核心转发逻辑)

```javascript
ProxyHandler
├── callOpenAIWithRetry()      // 带重试机制的 API 调用
│   ├── 最多重试 3 次
│   ├── 指数退避延迟
│   ├── 跳过某些错误重试
│   └── 最后一个错误
│
└── handleProxyRequest()       // 完整请求处理流程
    ├── 缓存检查
    ├── 请求记录
    ├── 获取 API 密钥
    ├── API 调用 (带重试)
    ├── 响应处理
    ├── 缓存存储
    └── 返回结果
```

## 缓存机制

### 缓存键生成

```
输入: prompt="猫咪" + size="1024x1024" + model="dall-e-3"
     ↓
MD5 哈希
     ↓
输出: "a7f3b8c1e2d9f4g6h5i3j2k1l9m0n7o8"
```

### 缓存生命周期

```
请求 1 (时间: T0, prompt="猫咪")
  ↓
缓存命中率: 0%
  ↓
调用 OpenAI API
  ↓
存入缓存 (过期时间: T0 + 1小时)
  ↓
请求 2 (时间: T0 + 10分钟, prompt="猫咪")
  ↓
缓存命中率: 50%
  ↓
从缓存返回 (节省 API 调用!)
  ↓
请求 3 (时间: T0 + 1小时 + 1分钟, prompt="猫咪")
  ↓
缓存已过期
  ↓
再次调用 OpenAI API
  ↓
更新缓存
```

## 密钥轮转机制

### 负载均衡

```
密钥配置: key1, key2, key3

请求序列:
1️⃣ Request → key1 ✓ (currentIndex = 0 → 1)
2️⃣ Request → key2 ✓ (currentIndex = 1 → 2)
3️⃣ Request → key3 ✓ (currentIndex = 2 → 0)
4️⃣ Request → key1 ✓ (循环回到开始)

好处:
- 分散请求负载
- 防止单个密钥被限流
- 提高整体吞吐量
```

## 错误重试机制

### 指数退避重试

```
请求失败 → 重试策略
           ↓
       重试 1 次
       等待时间: 2^(1-1) × 1000 = 1000ms (1秒)
           ↓
       仍然失败?
           ↓
       重试 2 次
       等待时间: 2^(2-1) × 1000 = 2000ms (2秒)
           ↓
       仍然失败?
           ↓
       重试 3 次
       等待时间: 2^(3-1) × 1000 = 4000ms (4秒)
           ↓
       仍然失败?
           ↓
       返回错误给客户端

总等待时间: 1 + 2 + 4 = 7 秒
最大请求次数: 4 次 (1次初始 + 3次重试)
```

## 性能优化建议

### 1. 使用 Redis 替换内存缓存
```javascript
// 生产环境建议
import redis from 'redis';
const client = redis.createClient();

// 缓存存储
await client.setex(cacheKey, 3600, JSON.stringify(data));

// 缓存获取
const cached = await client.get(cacheKey);
```

### 2. 添加请求速率限制
```javascript
import rateLimit from 'express-rate-limit';

const limiter = rateLimit({
  windowMs: 60 * 1000,    // 1分钟
  max: 10,                 // 最多10个请求
  message: '请求过于频繁，请稍后再试'
});

app.post('/api/images/generate', limiter, handler);
```

### 3. 使用数据库持久化日志
```javascript
// 改进的请求日志
await db.requestLog.insert({
  timestamp: new Date(),
  prompt: options.prompt,
  size: options.size,
  model: model,
  status: 'success',
  responseTime: endTime - startTime
});
```
